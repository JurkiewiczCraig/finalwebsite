Why Dashboard?

Dashboards are composed of configurable elements, such as maps, lists, charts, gauges, indicators, and tables. Most elements are data driven and thus makes it ideal to showcase the walkability index factors in a way that the intended audience can understand.
Positives Negatives Future Iterations
Easy to use the tools and attributes to show data in an intresting way Limited Customization which further requires Arcade coding Can work on live data to showcase interesting indicators.
Customization of layouts and themes Requires time to explore complex configurations Can explore a lot with the help of ArcGIS Arcade coding
Accessibility from any device Not available without internet connection
Integration with other ArcGIS tools and data sources Needs subscription/licence
Ability to integrate real-time data Performance suffers when large datasets are used

What was interesting?

The most interesting part to work with on this dashboard was color-coding the text in the list with the colors assigned for the respective walkability classification using Arcade coding.
The dashboard was designed for mobile devices too and has been arranged with the necessary parameters.
